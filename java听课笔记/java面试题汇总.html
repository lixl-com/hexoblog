多线程与并发

并发与并行的区别

都是同一时刻处理多任务的方式，

区别在于

并发：一个cpu通过切分时间片的方式，交替运行多个任务，让多个任务看起来同时运行。

并行：基于多cpu同时运行多个任务

进程与线程

进程：是资源分配的基本单位(按进程分配内存)

线程：是程序运行的基本单元(cpu按分配时间片的方式执行线程)

每个进程默认有一个线程，

多个线程之间共享所属进程的内存空间，

多个进程间通过IPC(socket/共享内存/共享文件等方式)进行通信。

线程创建的方式

1.继承Thread类，重写run方法（native start0方法）

2.实现Runnable接口及run方法，调用Thread类的start方法

3.实现FutureTask(Callable call())接口

线程间通信方式

1.共享变量(synchronized/lock)

2.管道通信(PipedOutputStream/PipedInputStream)

3.信号量(Semaphore)

4.条件变量(Lock.newCondition signal/await)

  1. 线程池参数有哪些，如何设置？
    核心线程数：cpu密级型(cpu+1)、IO密集型(cpu*2)
    最大线程数：可以创建的最大线程数
    非核心线程空闲时间(单位): 空闲时间超过该时间的线程会被销毁(直到等于设置的核心线程数)
    线程工厂：ThreadFactory，创建线程
    阻塞队列:
    拒绝策略：实现RejectedExecutionHandler接口，超出阻塞队列的任务会调用该该拒绝策略。
    主要有4种拒绝策略：

    1. AbortPolicy：直接丢弃任务，抛出异常，这是默认策略
    2. CallerRunsPolicy：让调用者所在的线程来处理任务
    3. DiscardOldestPolicy：丢弃等待队列中最旧的任务，并执行当前任务
    4. DiscardPolicy：直接丢弃任务，也不抛出异常

rdms

oracle与mysql的区别

1.mysql存储引擎(innodb/myisam/memory)，oracle支持事务 默认隔离级别不同(Read-Commited)

2.并发性能不一样，oracle锁粒度更小且不依赖索引；Mysql有可能锁定更长时间

3.安全性，oracle的管理员与普通用户分离，mysql都在一张表里

4.逻辑备份，Mysql备份是要锁定数据影响可用性

5.性能诊断工具，除了慢SQL查询，oracle更丰富一些；

聚簇索引和非聚簇索引

索引的数据结构是B+Tree，叶子节点用来存储数据(如主键索引)；

非聚簇索引，也叫二级索引，指向主键值而不是行数据(需要再次通过主键索引才能定位到行数据)
